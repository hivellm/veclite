# hivellm-veclite-ffi

C ABI for [VecLite](https://github.com/hivellm/veclite) — an embedded,
single-file, in-process vector database.

This crate is the stable C surface the language bindings are built on (Go, C#,
and any other FFI consumer). If you are writing Rust, depend on
[`hivellm-veclite`](https://crates.io/crates/hivellm-veclite) directly instead.

## Building

```sh
cargo build -p hivellm-veclite-ffi --release
```

Produces a `cdylib` and a `staticlib` named `veclite_ffi`. The matching
`veclite.h` header is generated by `cbindgen` and kept under a golden-file drift
test, so the committed header can never drift from the exported symbols.

## ABI

Every entry point returns an `int32` status code: `0` on success, a negative
`VL_ERR_*` value on failure. The error-code mapping is exhaustive over the
library's error enum — adding a variant without assigning a code fails the
build.

Each entry point wraps its body in `catch_unwind`, so a Rust panic surfaces as
`VL_ERR_INTERNAL` rather than unwinding across the FFI boundary. Retrieve the
message for the last failure on the calling thread with
`vl_last_error_message`.

Handles (`vl_db`, `vl_collection`) are opaque pointers and must be released with
their `_free` function; buffers returned to the caller are released with
`vl_buf_free`. Use `vl_abi_version` to check compatibility at load time.

## Features

- `onnx` — forwards the core crate's opt-in dense-embedding tier. Off by
  default so the shipped base library stays lean and network-free.
