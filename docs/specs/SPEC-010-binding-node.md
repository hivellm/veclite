Requirement IDs `NODE-xxx`. Binds the Rust crate directly via **napi-rs**. Conformance corpus (SPEC-015 §3) is the behavioral arbiter.

## 1. Packaging

- **NODE-001** Package `veclite`, Node ≥ 18 (also Bun; Deno via npm compat). Native prebuilds ship as per-platform `optionalDependencies` (`@veclite/darwin-arm64`, `@veclite/linux-x64-gnu`, `@veclite/linux-x64-musl`, `@veclite/win32-x64`, …) covering the FR-66 matrix — `npm install veclite` never compiles Rust.
- **NODE-002** ONNX ships separately as `@veclite/onnx` (EMB-040). WASM is a different package (`@veclite/wasm`, SPEC-012) — the native binding MUST NOT silently fall back to WASM.
- **NODE-003** TypeScript definitions generated by napi-rs and shipped in the package; the `.d.ts` is part of the conformance review.

## 2. API surface

```ts
import { open, memory } from "veclite";

const db = await open("app.veclite", { readOnly: false, durability: "normal" });
const docs = await db.createCollection("docs", {
  dimension: 384, metric: "cosine",
  hnsw: { m: 16, efConstruction: 200, efSearch: 100 },
  quantization: { scalar: { bits: 8 } },
  payloadIndexes: [["lang", "keyword"]],
});
const notes = await db.createCollection("notes", { autoEmbed: "bm25", dimension: 512 });

await docs.upsert("id-1", vector, { lang: "en" });     // vector: Float32Array | number[]
await docs.upsertBatch(points);
await notes.upsertText("id-9", "text…", { src: "readme" });

const hits = await docs.search(query, { limit: 10, efSearch: 200,
  filter: { must: [{ key: "lang", match: "en" }] }, withPayload: true });
const hits2 = await notes.searchText("query", { limit: 5 });
const hits3 = await docs.hybridSearch({ dense: vec, sparse: { indices, values }, alpha: 0.5, limit: 10 });
const page  = await docs.scroll({ limit: 100, offsetId: "id-500" });

await db.snapshot("backup.veclite"); await db.vacuum(); await db.close();

// sync twins for CLIs/scripts
const db2 = openSync("app.veclite");
const hitsSync = docs.searchSync(query, { limit: 10 });
```

- **NODE-010** Naming: camelCase, method-for-method with SPEC-004. Options objects mirror `OpenOptions`/`CollectionOptions`/query builder fields; absent fields take SPEC-004 defaults.
- **NODE-011** **Async by default**: every operation that can touch I/O or take > ~100 µs runs as a napi `AsyncTask` on the libuv threadpool — the event loop is never blocked. Every async method has a `*Sync` twin.
- **NODE-012** `Float32Array` crosses zero-copy in (`search`, `upsert`, `upsertBatch` with a flat `(n × dim)` array + `dimension`); hit vectors return as `Float32Array` views on the result buffer, detached when the result object is garbage-collected (external-buffer finalizer).
- **NODE-013** Handle lifecycle: `db.close()` idempotent, returns a promise that resolves after checkpoint; pending async operations settle (with `Closed` rejection for those not yet started). Finalizers close leaked handles with a `process.emitWarning`.

## 3. Errors

- **NODE-020** All failures reject (or throw, in `*Sync`) with `VecLiteError extends Error`: `code` (string, e.g. `"DIMENSION_MISMATCH"` — mirrors FFI constants without the `VL_ERR_` prefix), `message` identical to the Rust display string. Subclass-per-variant is NOT used (idiomatic Node: one error class + `code`).

## 4. Acceptance criteria

1. Conformance corpus green on the full prebuild matrix, Node 18/20/22 + Bun (gate G4).
2. Clean-machine test: `npm install veclite` + quickstart with no Rust toolchain.
3. Event-loop-liveness test: a 10 s bulk index keeps a concurrent timer ticking at ≤ 5 ms jitter (NODE-011).
4. Zero-copy proof for `Float32Array` paths (allocation tracking).
5. TypeScript: `tsc --strict` compiles the quickstart against the shipped `.d.ts`.
