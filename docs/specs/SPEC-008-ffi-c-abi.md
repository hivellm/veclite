Requirement IDs `FFI-xxx`. This ABI is the contract for the Go and C# bindings (SPEC-011) and any community binding; Python/Node bind the Rust crate directly (PyO3/napi-rs) but MUST expose the same behavior (conformance corpus).

## 1. General rules

- **FFI-001** Handle-based: opaque `vl_db*`, `vl_collection*`, `vl_hits*`, `vl_page*`. Handles are thread-safe (core is `Send + Sync`); any thread may call any function on any handle.
- **FFI-002** Every fallible function returns `int32_t`: `0` = OK; negative = error code (В§3). Results come back via out-params.
- **FFI-003** `catch_unwind` wraps **every** entry point; a panic maps to `VL_ERR_INTERNAL` and sets the thread-local error message. No unwinding across the boundary, ever.
- **FFI-004** Strings are UTF-8, NUL-terminated on input; library-returned buffers/objects are freed **only** by the matching `vl_*_free` function. Double-free and use-after-free are caller bugs but MUST NOT be exploitable via handle re-validation (generation-tagged handles SHOULD be used in debug builds).
- **FFI-005** Vectors cross as `(const float*, size_t len)`. Structured data (payloads, filters, points, options) crosses as bytes with a codec flag: `VL_CODEC_JSON` or `VL_CODEC_MSGPACK` (MessagePack reuses the exact struct shapes of the Vectorizer SDK wire types where they overlap).
- **FFI-006** The header is generated by `cbindgen` in CI and committed; a golden-file test fails the build if the generated header drifts from the committed one without a version bump.
- **FFI-007** Versioning: `vl_version()` (crate semver string), `vl_abi_version()` (u32, bumped only on additive changes; loaders gate on в‰¤), `vl_format_version()` (storage format). Symbols are never removed or re-typed within a major.

- **FFI-010** `vl_hit_view` exposes borrowed pointers (`id`, `score`, `payload bytes`, optional `vector`) valid until `vl_hits_free` вЂ” zero-copy for readers.
- **FFI-011** `query_opts` (MessagePack/JSON): `{ ef_search?, filter?, with_payload?, with_vector? }`; absent fields take SPEC-004 defaults. Filter document shape is SPEC-006.
- **FFI-012** `register_embedder` is deliberately **not** in the C ABI v1 (no callbacks вЂ” planning rule); custom providers are a Rust/PyO3/napi-level feature.

## 4. Packaging

- **FFI-030** `veclite-ffi` builds as `cdylib` + `staticlib`; release artifacts `libveclite.{so,dylib}` / `veclite.dll` / `libveclite.a` + `veclite.h` for the FR-66 platform matrix, published on GitHub releases (SPEC-016).

## 5. Acceptance criteria

1. Header golden-file test (FFI-006) in CI.
2. Panic-injection tests: every entry point forced to panic internally в†’ `VL_ERR_INTERNAL`, message set, process healthy (T4.2).
3. Error-code table exhaustiveness: compile-time match on `VecLiteError` variants в†” codes (adding a variant without a code fails the build).
4. Valgrind/ASan run of the C smoke test (open в†’ create в†’ upsert_batch в†’ search в†’ scroll в†’ close) вЂ” zero leaks, zero invalid reads.
5. Concurrency smoke: 16 threads hammering one `vl_db*` across all ops under TSan.
